Collating

Collating refers to combining a batch of observations so that the arrays in individual observations are stacked together . As an example, consider a dataset with 100 observations, each with 2 features .


			
			
			
			
			data
			
			 
			
			= 
			
			
			rand
			
			(
			
			2
			
			, 
			
			100
			
			)  # observation dimension is last by default

We can collect a batch of 4 observations into a vector as follows:


			
			
			
			
			using
			
			 
			
			
			



			DataLoaders
	
			
			

			
			
			
			
			batch
			
			 
			
			= 
			
			
			[
			
			
			
			getobs
			
			(
			
			data
			
			, 
			
			i
			
			) 
			
			
			for
			
			 
			
			
			
			i
			
			 
			
			in 
			
			
			1
			
			:
			
			4
			
			]

			
			4-element Vector{Vector{Float64}}:
 [0.19167812627953018, 0.34399417909374974]
 [0.35407797072269354, 0.15276535264616642]
 [0.4563622776477283, 0.9479604184109643]
 [0.17452576525416952, 0.5864718534374237]

Many machine learning models, however, expect an input in a collated format: instead of a nested vector of vectors, we need a single ND - array . DataLoaders . jl provides the collate function for this:


			
			
			
			



			DataLoaders
	
			
			.
			
			



			collate
	
			
			(
			
			batch
			
			)

			
			2×4 Matrix{Float64}:
 0.191678  0.354078  0.456362  0.174526
 0.343994  0.152765  0.94796   0.586472

As you can see, the batch dimension is the last one by default .

Nested observations

The above case only shows how to collate observations that each consist of a single array . In practice, however, observations will often consist of multiple variables like input features and a target variable . For example, we could have an integer indicating the class of an input sample .


			
			
			
			
			inputs
			
			 
			
			= 
			
			
			rand
			
			(
			
			2
			
			, 
			
			100
			
			)

			
			
			
			
			targets
			
			 
			
			= 
			
			
			rand
			
			(
			
			
			1
			
			:
			
			10
			
			, 
			
			100
			
			)

			
			
			
			
			data
			
			 
			
			= 
			
			
			(
			
			inputs
			
			, 
			
			targets
			
			)

			
			
			
			
			batch
			
			 
			
			= 
			
			
			[
			
			
			
			getobs
			
			(
			
			data
			
			, 
			
			i
			
			) 
			
			
			for
			
			 
			
			
			
			i
			
			 
			
			in 
			
			
			1
			
			:
			
			4
			
			]

			
			4-element Vector{Tuple{Vector{Float64}, Int64}}:
 ([0.7687327312142116, 0.05328592089659989], 6)
 ([0.6554625901480687, 0.7150455909817023], 7)
 ([0.37560950957941186, 0.3443496550225855], 5)
 ([0.567031991869908, 0.5355989676232619], 5)

Collating here also works, by keeping the tuple structure and collating each element separately:


			
			
			
			



			DataLoaders
	
			
			.
			
			



			collate
	
			
			(
			
			batch
			
			)

			
			([0.7687327312142116 0.6554625901480687 0.37560950957941186 0.567031991869908; 0.05328592089659989 0.7150455909817023 0.3443496550225855 0.5355989676232619], [6, 7, 5, 5])

This is also implemented for NamedTuple s and Dict s . You can also collate nested structures, e . g . a Tuple of Dict s and the structure is preserved . This also works when using inplace loading .
